# 通用识别使用文档 —— generalOCR

## 使用说明

###### 运行服务器docker镜像：使用命令`docker run -it -p 5000:5000 <new_image_name> `

![截图](ec1d97d318b88699bb75faccb39eb8bb.png)
                                                                                                                         (此处IMAGE ID可能有变化建议使用image_name)

有两种使用方式：

1.调用API接口输入单张发票照片路径还有需要识别的项目名称

2.在一个文件夹里放置任意张待识别发票照片，输出结果自动生成在一张csv表格里包括所有项目类别的结果

<br/>

#### API接口调用说明（使用方法1）

在docker内使用`python API_service.py` 启动Flask服务

在外部使用命令`curl -X POST -F 'image=@path/to/your/image.png' 'http://<服务器IP>:5000/ocr?category=<提取项目名>'`返回json格式识别结果

> Ex：
> 
> ```
> Input: `(base) [deepbase@deepbase invoiceOCR]$ curl -X POST -F 'image=@images/23.png' 'http://0.0.0.0:5000/ocr?category=购买方纳税人识别号'`
> 
> Output: `{"购买方纳税人识别号": "\"310114772120643\""}`
> ```

目前支持的提取项目有：

1.发票代码 2.发票号码 3.开票日期 4.效验码 5.税率 6.价税合计(小写) 7.购买方名称 8.销售方名称

9.购买方纳税人识别号 10.销售方纳税人识别号 11.购买方开户行及账号 12.销售方开户行及账号

<br/>

#### 批量图片处理和表格自动化生成说明 （使用方法2）

在docker里的根路径下可以看到images这个文件夹，将要识别的发票图片(jpg, png)格式全部放进去

使用命令`python fpocr.py`启动程序，它会识别所有images文件夹里的图片并将最终生成的识别结果保存在"识别结果.csv"里并保存在images文件夹中

<br/>

<u>图像预处理：</u>

首先用open cv去检测图片是否是摆正的，如果没摆正就调用rotate_image对它旋转一定角度。

经过实验我发现旋转2次可以让所有角度拍摄的图片都回正所以我写了correct_image_orientation_twice这个函数

> ```
> rotate_image(image, angle) #根据指定的角度旋转图像
> 
> correct_image_orientation(image) #将图像转换为灰度图像，然后使用Canny边缘检测和霍夫变换检测直线，从而校正图像方向。
> 
> correct_image_orientation_twice(image_path) #读取图像并调用两次 correct_image_orientation 函数，以提高校正精度。
> ```

<br/>

<img src="a80f4c1aee2593aecc3d20ede4ce148b.jpg" alt="db8dcf436984f2004b7a8e0004319d4.jpg" style="zoom:20%;" />

从而导致识别率非常低下,所以我在之后fp_ocr函数中首先执行一次识别如果其平均置信度过低低于0.96那么证明这张照片是反过来             拍的就将其rotate180度回正再进行二次识别

```
confidence = ocr_confidence(result)
    if confidence < 0.96:
        # Rotate the image 180 degrees and re-run OCR
        corrected_image = rotate_image(corrected_image, 180)
        result = ocr.ocr(corrected_image, cls=True)
```

（optional）: 你可以在 corrected_image 这个文件夹里查看预处理过的发票照片

<br/>

<u>辅助函数</u>

get_specific_data(image, category): 调用fp_ocr函数对图像进行OCR处理，并返回指定类别的数据。如果指定的类别在OCR结果中不存在，则返回'Category not found'。如果要修改API访问返回格式可以在此更改

```
def get_specific_data(image, category):
    data = fp_ocr(image)
    return {category: data.get(category, 'Category not found')}
```

<u>处理请求</u>

> @API_service.route('/ocr', methods=['POST']): 定义一个路由，当收到POST请求时调用ocr_endpoint函数。
if 'image' not in request.files: 检查请求中是否包含图像文件，如果没有，返回400错误响应。
file = request.files['image']: 获取图像文件。
if file.filename == '':: 检查文件名是否为空，如果是，返回400错误响应。
category = request.args.get('category', 'text'): 获取URL参数中的类别参数，如果没有提供，默认为'text'
category = category.encode('latin1').decode('utf-8'): 假设类别参数是UTF-8编码的，将其解码为UTF-8字符串。
print("Decoded category:", category): 打印解码后的类别参数。
image = np.frombuffer(file.read(), np.uint8): 将图像文件读取为字节数组，并转换为NumPy数组。
image = cv2.imdecode(image, cv2.IMREAD_COLOR): 使用OpenCV解码图像。
result = get_specific_data(image, category): 获取指定类别的OCR结果。
print("OCR result:", result): 打印OCR结果。
response = make_response(json.dumps(result, ensure_ascii=False)): 创建JSON响应，确保非ASCII字符不被转义。
response.headers['Content-Type'] = 'application/json; charset=utf-8': 设置响应头，指定字符集为UTF-8。
return response: 返回响应。
